    <h2>Overview</h2>
      <div className="columns">
        <div className="column">
          <p>Dialogs are components used to display content in a secondary window that appears in a layer above the main window. It should always be triggered by a user's action, not surprising by popping up.</p>
          <p>It might be used for messages that should be read with attention and, sometimes, blocks the interaction with the rest of the page. In this case, this type of component interrupts user experience flow on the underlying page and it could be perceived as annoying if it's not used when really necessary. We can have 2 types of Dialogs:</p>
          <p><strong>Modal</strong>
          <br/>
          When active, blocks the user's workflow untill the dialog modal task is completed or dismissed. The underlying page is obscured to show its block.</p>
          <p><strong>Modeless</strong>
          <br/>
          Is a dialog used for elective tasks related to the page main context, but it doesn't block the on-page content.</p>
        <div className="doc-em-block">
          <p>Currently, we only use modal dialogs in our apps. Nevertheless, it's important to know that modeless dialogs are more recommended and should be considered for further contexts. We can add them in the future as needed.</p>
        </div>
      </div>
      <div className="column">
        <p>Currently there are <strong>two variations of dialogs</strong>:</p>
        <p><strong>Standard Dialog</strong>
        <br/>
        Standard Dialog is used to show brief content that is usually more important than the one in the page and sometimes presents a call to action that must be completed to close the modal.</p>
        <p><strong>Message Dialog</strong>
        <br/>
        This component is often used to communicate a short message or to  confirm an action a user has taken. Normally, contains just a single button, and an icon to illustrate the message.</p>

    <h2>Behavior</h2>
    <div className="columns">
      <div className="column">
        <p><strong>MODAL</strong></p>
        <p><strong>Focus</strong></p>
        <p>When a modal is opened, a curtain overlay (Black 50) should cover the on-page content, providing focus to the content showed in the dialog. This also indicate to the user that he can't interact with the underlying page.</p>
        <p><strong>Dismissing</strong></p>
        <p>By default, modals can be dismissed by clicking on the “x” in the right corner of the container. It also can be closed by clicking in one of the call to action buttons (if the modal have some). ‘Esc’ key should close the window, the same as a click outside the modal.</p>
        <p><strong>MODELESS</strong></p>
        <p><strong>Focus</strong></p>
        <p>When a dialog opens, initial focus should stay on the first element that allows user input. Despite that, the underlying page still remains free to interact.</p>
        <p><strong>Dismissing</strong></p>
        <p>Modeless dialogs can be dismissed by clicking on the “x” in the right corner of the container. ‘Esc’ key should close the window as well.</p>
      </div>
      <div className="column">
        <p><strong>MOBILE</strong></p>
        <p><em>Message Dialogs behaves on mobile just as the desktop versions of its modals, except the it doesn't support the "esc” keyboard interaction. A slightly difference of interaction will happen on full-width version as the following text explicits.</em></p>
        <p><strong>Focus</strong></p>
        <p>When a modal is opened, a curtain overlay (Black 50) should cover the on-page content, and the container should appears in a animation from bottom to top (0.25s) providing focus to the content showed in the dialog. This also indicate to the user that he can't interact with the underlying page.</p>
        <p><strong>Dismissing</strong></p>
        <p>The full-width mobile modal can be closed by swiping the container to the bottom, as the arrow icon at the top suggests. The interaction of the dismissing follows the swiping flow.</p>
        <div className="doc-em-block">
          <p>If the mobile modal content doesn't fit the modal height, then the section should scroll vertically. Depending of the content, the designer should pick what elements will remain fixed, with no scrolling.</p>
        </div>

To open the hidden content, the user should swipe up to enlarge the dialog. To dismiss, the user can either click outside the component, or swipe down, as the icon suggests.</p>
