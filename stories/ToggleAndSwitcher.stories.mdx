# Toggle & Switcher

The `Toggle` component might receive up to five inputs as props, where the `color` and `name` parameters are the only required ones.
- `name`: should be an unique identifier and works as the `id` of the `input` tag within the component; it's also an important accessibility tool
- `color`: defines the Toggle color
- `disabled`: sets the Toggle component as disabled; includes functionality and styles
- `defaultValue`: you can set a default value for the toggle to start with; it normally starts as `false`
- `value`: you might need to set the component value from outside the component; if so, the internal logic to handle the component value won't be used
- `handleToggle`: you might need to perform other actions when the component value changes; use this param to pass a function that's gonna be called on value changes
