# Tag

The `Tag` component might receive up to five inputs as props, where the `color` and `name` parameters are the only required ones.
- `name`: should be an unique identifier and works as the `id` of the `input` tag within the component; it's also an important accessibility tool
- `color`: defines the Tag color
- `disabled`: sets the Tag component as disabled; includes functionality and styles
- `defaultValue`: you can set a default value for the Tag to start with; it normally starts as `false`
- `value`: you might need to set the component value from outside the component; if so, the internal logic to handle the component value won't be used
- `handleTag`: you might need to perform other actions when the component value changes; use this param to pass a function that's gonna be called on value changes
