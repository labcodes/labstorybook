  <h2>Button positioning and order</h2>
    <p><code>Button</code> always appear on the right of modals, dialogs, forms and most of UI parts. <code>OutlineButton</code> and <code>TextButton</code> appear and pile to the left of <code>Button</code>.</p>
    <ul>
      <li>Use right-aligned buttons primarily and if the main response advances the user through a screenflow (from left to the right).</li>
      <li>Use left-aligned buttons to place actions at bottom of lists, comparative collums, tabled layouts, multiple content collumned layouts.</li>
      <li>Use full-width buttons for dialogs and side panels.</li>
    </ul>

  <h2>Labels</h2>
  <p>Button labels must say what will happen when the user interacts with it.</p>
  <ul>
    <li>Use sentence case (only the first word in a phrase and any proper nouns capitalized)</li>
    <li>Make them as short as possible (max. of 3 words)</li>
    <li>Clearly explain what happens when the button is clicked.</li>
  </ul>
  <h2>Icon use</h2>
  <ul>
    <li>Use an icon to add meaning to the button's action, function or to indicate triggers as "open external link".</li>
    <li>The icon must be directly related to the action that the user is taking. <span class="u-color-cyan-60">Don't hesitate to create a new icon when it is necessary.</span></li>
    <li>Use icon frames of 16px within buttons.</li>
    <li>Icons should always appear to the left of the label.</li>
    <li>Always use the same color value as the label within a button. Icon follows label with equal emphasys.</li>
  </ul>
  <p><span class="u-color-pink-60">An only icon button is represented by a pictogram. Use it on extreme situations. Make sure the drawing is clear and communicate in one glance what action it triggers.</span></p>
  </div>
