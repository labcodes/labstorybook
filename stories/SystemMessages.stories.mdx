  <h2>Overview</h2>
  <p>System messages can be divided into 2 subgroups: Banners and Alerts. </p>
  <div className="columns">
    <div className="column">
      <p><strong>Banners:</strong> persistent, prominent messages about system-wide states. They persist during the whole user session, not being dismissible through buttons, except the informational variant.</p>
      <p>Banners do not overlay any content but push down the whole page or frame content. They appear only one at a time and only disappear if or when the related issue is resolved. Use banners to communicate errors, warnings, and information with their associated options or actions. Banners contain a single message and specific actions a user can take.</p>
    </div>
    <div className="column">
      <p><strong>Alerts:</strong> persistent alerts that show specific information, warnings, or errors. They appear into the specific div or frame the closest possible to the related issue or component. The informational variant is dismissible.</p>
    </div>
  </div>

  <h3>Elements and Format</h3>
  <div className="columns">
    <div className="column">
      <p><strong>Icon (mandatory for tablet-up):</strong> They bring clarity to the message. Icons should be placed to the left of a title with a frame size of 24px. The icon must be directly connected to the message meaning. Never use icons for alerts in a mobile environment.</p>
      <p><strong>Message (mandatory):</strong> The alert message should be short and include: a description of the issue, what is affected, and what to do to fix the situation.</p>
    </div>
    <div className="column">
      <p><strong>Button (optional):</strong> Text button.</p>
      <div className="doc-em-block"><p>You must always tell the user what <strong>action</strong> is needed to take place, so the issue presented is solved.</p></div>
    </div>
  </div>

  <h2>Behavior</h2>
  <div className="columns">
    <div className="column">
      <p><strong>Banners</strong></p>
      <p>As a system wide communication component, they appear at the very top of the app pushing down all the content by animation (600ms), staying fixed on this position even if the user scrolls the page.</p>
    </div>
    <div className="column">
      <p><strong>Alerts</strong></p>
      <p>These appear inside a frame or div of a specific set of components or function/feature, the closest to where the message is related, with a slide animation (600ms). The position (top, bottom, left of right) is free for the designer to choose and depends on the project settings and white space availability.</p>
      <p><strong>Piling:</strong> It is not possible to pile banners. Inline alerts can be pilled up to 3 components, but avoid doing so.</p>
    </div>
  </div>

  <h2>Messages</h2>
  <p>Messages for alerts tell the user about important system or feature-related issues or status. It’s initiated by the system and it’s not a result of the user’s actions.</p>

  <div className="columns">
    <div className="column">
      <p>The alert message should be short and include: a description of the issue, what is affected and what to do to fix the situation.</p>
      <p><strong>Content:</strong> never put the blame on the user for causing any issue, but let  them know exactly what's happening by being clear and conversational.</p>
      <p><strong>Lenght:</strong> try to be concise and use the least number of words possible.</p>
    </div>
    <div className="column">
      <div className="doc-em-block">
        <p><strong>For tablet-up</strong> alerts never write a message that uses more than <strong>2 lines</strong>.</p>
        <p><strong>For mobile</strong>, never more than <strong>3 lines</strong>.</p>
        <p>Remember that banners are resizable, and, with that, message's size shifts up and down. Pay attention to the space the message element occupies to never exceed the limits above.</p>
      </div>
    </div>
  </div>

  <h3>Padding and Spacing</h3>
  <p><strong>Banners</strong> have specific rules for paddings and margins. As they fill the screen width completely, there are some spacing rules that need to be know before using these components. Both tablet-up and mobile variations have 3 types and rules for spacing: vertical, horizontal, and inner.</p>

  <div className="doc-container">
    <img src="docs/system-messages/padding.png" alt="Paddings, margins and spaces for banners" />
    <img src="docs/system-messages/padding-mobile.png" alt="Paddings, margins and spaces for mobile banners" />
  </div>

  <p><strong className="doc-pink">Inner spacing (between elemens):</strong> fixed. On tablet-up: 16px / On mobile: 4px</p>
  <p><strong className="doc-orange">Vertical padding (top and bottom):</strong> fixed value of 12px</p>
  <p><strong className="doc-cyan">Horizontal padding (left and right):</strong> dynamic for tablet-up and fixed on mobile</p>

  <div className="columns">
    <div className="column">
      <p><strong>Tablet-up:</strong> as inner elements are centered, the value is dynamic, but the minimum is equal the margin size of the used viewport grid.</p>
